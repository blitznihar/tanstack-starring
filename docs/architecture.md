# Comet Academy Architecture

## Overview

Comet Academy is a Bun and TanStack Start application with React routes, server functions, MongoDB repositories, pure domain modules, and optional integrations for OpenAI, Auth0, Stripe, SMTP, New Relic, Docker, Vercel, and Electron.

## Runtime Flow

```text
Browser route
  -> TanStack route loader/action
  -> src/server/rpc server function
  -> src/server service
  -> src/repositories repository
  -> MongoDB
```

Pure calculations should flow through:

```text
route/server
  -> src/domain pure function
  -> tested output
```

## Key Directories

### `src/routes`

File-based TanStack Router routes. Route components define:

- search param validation
- loader dependencies
- server-function calls
- page-level UI

Important routes include:

- `/student`
- `/history`
- `/lesson`
- `/practice`
- `/exam/$sessionId`
- `/dashboard`
- `/scoring`
- `/billing`
- `/admin/console`
- `/admin/content`
- `/admin/rewards`
- `/admin/scheduler`
- `/admin/profile`
- `/health`

### `src/server/rpc`

Browser-callable server functions. These must validate input, call `requireAuth()` where appropriate, enforce authorization, and return safe DTOs.

### `src/server`

Server-side business workflows and integrations:

- auth
- billing
- content
- exam
- gamification
- mastery
- notifications
- observability
- practice
- profile import/export
- programs
- reporting
- rewards
- scheduler
- scoring
- users

### `src/repositories`

MongoDB access layer. Code outside this directory should not import MongoDB driver primitives for app data access.

### `src/domain`

Pure, unit-tested business logic:

- billing access/pricing
- content validation
- conversion
- exam assembly/session/scoring
- ledger
- mastery
- pools
- practice assembly/awards
- profile last-write-wins logic
- prompt generation helpers
- rewards calculations
- scheduler
- scoring

### `src/schemas`

Zod schemas for persisted, imported, and exported data.

### `content`

Authored/importable content bundles.

### `scripts`

Operational scripts for seed/import/export/user creation/demo.

### `electron`

Optional desktop shell and app icon packaging.

## Data Model Concepts

- User: authenticated account and role metadata.
- Program: a learning program with subjects, config, Robux rules, schedule rules, and concepts.
- Enrollment: student-to-program assignment.
- Schedule: enrollment-specific ordered work plan.
- Lesson progress: completed lesson standard for an enrollment.
- Practice progress: completed practice standard for an enrollment.
- Response: persisted answer attempt.
- Robux ledger: idempotent earn/penalty records.
- Reward rule: configured target and prize.
- Redemption: requested reward fulfillment.
- Exam session: exam state and responses.
- Scoring job: async/manual scoring queue for written responses.

## Environment and Deployment

- Local and preview DB: `comet-dev` unless `MONGODB_DATABASE` overrides it.
- Production DB: `comet`.
- Vercel preview deploys validate changes before approval-gated production.
- Docker can run local MongoDB and app services.
- Electron can point to local or network app URLs.

## Server-Only Packages

The Vite config stubs server-only packages in browser builds. Keep these packages server-only:

- `mongodb`
- `argon2`
- `newrelic`
- `stripe`

## Generated Files

`src/routeTree.gen.ts` is generated by TanStack Router. Add route files and run `bun run build` instead of hand-editing route tree output.

## Observability

- `/health` is safe for synthetic monitoring.
- New Relic is opt-in through environment variables.
- Useful dashboards:
  - error rate
  - response time
  - throughput
  - OpenAI scoring failures/timeouts
  - MongoDB latency
  - SMTP failures
  - Auth0 callback failures
  - Stripe checkout/webhook failures
